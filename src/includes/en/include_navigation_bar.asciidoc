With the _navigation bar_ on the left and the icons it contains, you make the basic decisions about what {CMK} should do for you:

The {CMK} logo -- for displaying the xref:intro_gui#start_page[start page].

[.guihint]#Search# -- for the xref:search[search].

[.guihint]#Monitor# -- for the monitoring itself.

[.guihint]#Customize# -- for the customization of interface elements useful for the monitoring such as bookmarks, views, dashboards, and graphs.

[.guihint]#Setup# -- for setting up the objects to be monitored (such as hosts and services).

Behind these three icons [.guihint]#Monitor#, [.guihint]#Customize#, and [.guihint]#Setup# are more or less extensive menus, the so-called 'mega menus' or icon menus, whose entries are divided into several topics
-- for example, in the [.guihint]#Setup# menu under the [.guihint]#Hosts# topic you will find entries for configuring hosts, host groups, host tags and host-specific rules.

[.guihint]#Changes# -- for xref:glossar#activate_changes[activating pending changes]. +
If there are pending changes, their number will be displayed in yellow in the [.guihint]#Changes# icon.

[.guihint]#Help# -- for displaying helpful information. +
At the top right of the [.guihint]#Help# menu, you will find the edition and version of {CMK} currently in use.
The menu itself contains entries with which you can call up documentation and information -- inside {CMK} or outside.
Among other things, you can also open this User Guide. +
ifdef::onprem[]
If there are incompatible changes after an xref:update#[update] of the site, you will see the number in red background color in the [.guihint]#Help# menu icon. 
endif::[]
ifdef::saas[]
If there are incompatible changes after an xref:admin_panel#site_update[update] of the site, you will see the number in red background color in the [.guihint]#Help# menu icon. 
endif::[]
In the [.guihint]#Help# menu itself, a link in red background color is then displayed with which you can open the list of xref:glossar#werk[Werks] of these incompatible changes. 

[.guihint]#User# -- for changing your user account. +
At the top right of the menu you will see the current username and the xref:wato_user#roles[role] assigned to this user. 
In this menu you can personalize your profile and log out of the {CMK} interface.
An overview of the available personal settings can be found in the chapter on the xref:user_interface#user_menu[User menu].

ifdef::saas[]
[.guihint]#Admin# -- for administering {CE}. +
This button opens the xref:admin_panel#[admin panel] in a new browser tab.
endif::[]

[.guihint]#Sidebar# -- for showing and hiding the xref:glossar#sidebar[sidebar].


[discrete]
==== Keyboard shortcuts

For quick navigation in the icon menus, various keyboard shortcuts are available.

[cols="20,~",options="header"]
|===
|Shortcut |Action
| kbd:[Strg+K] |Opens the search.
| kbd:[Alt+M] |Opens the [.guihint]#Monitor# menu.
| kbd:[Alt+C] |Opens the [.guihint]#Customize# menu.
| kbd:[Alt+S] |Opens the [.guihint]#Setup# menu.
| kbd:[Esc] |Closes the open menu.
| kbd:[Tab] |Next hit in the search results.
| kbd:[Shift+Tab] |Previous hit in the search results.
| kbd:[Enter] |Call up search hit.
|===

[TIP]
====
In the {CMK} GUI, JavaScript evaluates the number assigned to a key (_keycode_) and not the symbolic name derived from it (_keysym_).
The keys shown in the table therefore refer to the QWERTZ keyboard layout.
If you are working with a different layout (for example, AZERTY in the French-speaking parts of Switzerland), you may need to use other keys.
====
