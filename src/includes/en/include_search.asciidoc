The {CMK} interface has a central location for global searches:
the [.guihint]#Search# icon in the navigation bar.
When you click on [.guihint]#Search#, the input field for your search terms will be displayed, along with a few other elements to help you with your search:

.Before the search, {CMK} displays details of the latest search queries
image::gui_search.png[alt=“The search environment in {CMK}.”,width=89%]

Before you start searching, i.e. when the input field is empty, two lists show details of your last five searches.
Of course, this only applies if you have already searched a few times and have not deleted your search history with [.guihint]#Clear all#.
[.guihint]#Recently viewed# shows which pages were last opened from the search, and [.guihint]#Recently searched# shows the search terms entered.
You can click on any of the entries in the list to open the corresponding page immediately or to transfer the search term to the input field.

The fastest way to start is to type a search term into the input field.
You can use any keywords you like when entering your search.
You do not need to worry about upper and lower case letters.
When you finish your entry by pressing the kbd:[Enter] key, the search will close and on the main page {CMK} will display all matching hosts or services in the monitoring.

Starting the search quickly is not always the best way to achieve the desired result --
for example, if you are not searching for the name of a host or service, rather instead for a specific xref:glossar#rule[rule] or global setting.
Hosts and services are found in the xref:glossar#monitoring_environment[monitoring environment], rules and global settings in the xref:glossar#configuration_environment[configuration environment].
It is fortunate, then, that {CMK} searches in multiple environments.

As soon as you start typing your search term, you will immediately see the results below the search field, along with the areas in which they were found:

.Clicking on any of the entries in the list opens its corresponding page
image::gui_search_results.png[alt=“The results after entering a search term in the input field.”,width=89%]

Grouping improves the overview, especially when there are a large number of results.
Initially, only a maximum of five results are displayed per group.
You can then reload the rest using the last entry [.guihint]#Show all#.
The results are usually grouped according to the topics under which they can be found in the icon menus of the navigation bar.

{CMK} searches in the [.guihint]#Monitor#, [.guihint]#Setup#, and [.guihint]#Customize# menus of the navigation bar.
Below the input field, you have the option of restricting the scope of the search.
This is particularly useful if you already know in which area you expect to find the result, for example, a service in [.guihint]#Monitoring# or a rule in [.guihint]#Setup#.

With [.guihint]#All#, the search is performed in all three areas.
This is the default setting when you start the search using the [.guihint]#Search# icon.
However, you also have the option of starting the search in the [.guihint]#Monitor#, [.guihint]#Setup#, or [.guihint]#Customize# menu using the input field that appears at the top after opening the menu.
After entering the first character, you will be redirected to the global search.
The search area will be limited to the menu from which you started the search.

[{image-left}]
image::gui_search_results_edit.png[alt=“Search result containing a button for opening the host properties.”,width=390px]
In the search results, a line containing a host name sometimes also contains the icon:icon_edit[alt=“Icon for editing a list entry.”][.guihint]#Edit# button, which allows you to directly open and edit the properties of this host.
The results of the search in the configuration environment and monitoring environment are combined here.
Therefore, this button is only displayed if you have searched in the [.guihint]#All# area and the host exists in both the configuration environment _and_ the monitoring environment.

{CMK} performs the search in the background using various methods, collects the results, and presents them in a uniform manner in the global search.
