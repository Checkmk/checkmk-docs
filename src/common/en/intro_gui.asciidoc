[#start_page]
== The start page

This chapter takes a closer look at the parts of the {CMK} graphical user interface (GUI) that we only touched on in the xref:intro_setup_monitor#[previous chapter].

The start page is displayed after you log in.
By default, it shows a dashboard, the so-called [.guihint]#Main dashboard#, unless the welcome page is displayed.
How to change the start page to this default dashboard is described in the chapter on the xref:intro_welcome#[welcome page].

You can change the start page via your user profile, which can be accessed via the [.guihint]#User# menu in the navigation bar with [.guihint]#User > User profile > Edit profile#.
The option for the start page is called [.guihint]#Start URL to display in main frame#.
You can find details in the article on the xref:user_interface#user_menu[user interface].

In the following sections, you will learn about the basic elements of the interface, all of which can be seen in the image above.
The most important thing to note is the division into the *navigation bar* on the left, the *main page* in the middle, and the *sidebar* on the right.

[#main_page]
== The main page

The main page is the area between the xref:navigation_bar[navigation bar] on the left and the xref:sidebar[sidebar] on the right.
After logging in, you will see the xref:start_page[start page] displayed in this area.
What you see on the main page depends on where you are currently located in {CMK}.
The content of the main page changes depending on your selection in the navigation bar or the sidebar. 
For example, in the [.guihint]#Monitor# menu, you can select [.guihint]#Hosts > All hosts#:

.The main page with the view of all hosts
image::intro_monitor_all_hosts.png[alt=“The main page with the 'All hosts' view.”]

Above the page title, you will see the path to the current page, always starting with the name of the menu from the navigation bar.
This 'breadcrumb navigation' helps you to know where you are in {CMK} even after complex actions -- the above image showing the [.guihint]#All hosts# page in monitoring.

[#show_less_more]
== Show less or Show more

{CMK} provides a wide range of setting and selection options in many places on the interface.
This selection can sometimes be quite overwhelming, so that one can lose sight of the essentials and orientation becomes difficult -- not only, but especially for newcomers to monitoring.

As a measure to reduce complexity, you will find three dots (*...*) in many
locations on the {CMK} interface, such as in the [.guihint]#Overview# or here on the far
right in the first line of the [.guihint]#Monitor# menu:

.Three points (...) are used to reduce complexity in {CMK}.
image::intro_monitor_menu_showmore.png[alt="First line of the Monitor menu with search input field in Show less mode.",width=86%]

Whenever you see these ellipsis points, {CMK} offers two views:
In the 'Show less' mode only the most important entries are shown (meant for the beginner), and in the 'Show more' mode all entries (for the expert).
You can switch between the two modes by clicking on the omission points:
show more with icon:button_showmore[alt="Icon for switching to Show more mode."] and show less with icon:button_showless[alt="Icon for switching to Show less mode."].

You set the general behavior in the settings for your user profile ([.guihint]#User > Edit profile > Show more / Show less > Set custom show mode#) by setting one of the two modes as the default -- or do without both modes and face the unfiltered complexity with [.guihint]#Enforce show more#.
