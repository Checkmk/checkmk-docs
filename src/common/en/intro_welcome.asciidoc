[#welcome]
== The welcome page

So, {CMK} is ready.
After successfully logging in to {CMK} in the previous chapter, you will be greeted with a warm welcome to {CMK}.
Since version {v25}, the welcome page is the new starting point for newly created sites when a {CMK} administrator logs in.
The restriction to administrators is due to the fact that the actions that can be performed here require permissions that are usually only available to {CMK} administrators.

The welcome page anchors access to helpful information directly in the user interface.
It helps you quickly complete the most important configuration tasks in {CMK}, such as creating xref:glossar#host[hosts], customizing xref:glossar#service[services], or setting up xref:glossar#notification[notifications].

The list of [.guihint]#Topics to explore# serves this purpose.
The individual topics are structured in such a way that you can quickly find what you are looking for.
Each topic first provides brief introductory information and then gives specific instructions that you can then in a series of steps carry out directly in the GUI.
You can open the corresponding page in {CMK} directly in one step by clicking a button, without having to navigate manually through the GUI.

Once you have completed a topic, you can check it off yourself using the [.guihint]#Mark as done# button and continue with the next topic.
Each topic you mark as complete advances the progress bar at the top of the page indicating the topics you have explored.
However, you still have the option to return to steps you have already completed.

The very first item in [.guihint]#Topics to explore# explains how in just a few steps you can add the first host to your monitoring system.
Using {CMK} without hosts is only moderately exciting, because without anything to monitor, the monitoring interface remains empty and you can't really see what {CMK} is capable of.

The topics on the welcome page focus on the 'how' rather than the 'why'.
If you are looking for more detailed information, you will find it, as usual with {CMK}, in this User Guide and, in particular, in this Beginner's Guide.
Welcome page and Beginner's Guide have the same goal: to help you set up your monitoring.
In the next chapter of the Beginner's Guide, we will show you how to create folders and hosts.
That's a promise.

If you want to start with the first topic in [.guihint]#Topics to explore#, be sure to first read a few notes on how to use the welcome page.
If you have navigated to another page in {CMK}, there are several ways to return to the welcome page:

* On the left side of the {CMK} user interface, you will find the xref:intro_gui#navigation_bar[navigation bar].
Clicking on the {CMK} logo at the top of the navigation bar will take you back to the start page.
This start page is and will remain the welcome page unless you have changed it manually.

* You can always open the welcome page via the navigation bar, the [.guihint]#Help# menu, the topic [.guihint]#Learning {CMK}# and the entry [.guihint]#Welcome page# -- or more briefly via [.guihint]#Help > Learning {CMK} > Welcome page#.

* In the xref:intro_gui#sidebar[sidebar] on the right-hand side, you will find the xref:glossar#snapin[snap-in] [.guihint]#Welcome to {CMK}# at the top:
+
.The top snap-in takes you back to the topics on the welcome page
image::intro_snapin_welcome.png[alt=“The Welcome to {CMK} snap-in.”,width=50%]

Clicking on [.guihint]#Continue exploration# opens a so-called _slideout_.
A slideout temporarily displays information relevant to the current context.
A slideout slides in usually from the right side over the current page in {CMK}, which is then dimmed.
When you no longer need the slideout, you can simply close it again. +
The [.guihint]#Welcome to {CMK}# snap-in opens a slideout that displays the [.guihint]#Topics to explore#.
The first topic that has not yet been completed is displayed, meaning you can continue where you left off.
The slideout contains the [.guihint]#Open full view# button, which reopens the complete welcome page.

If you are confident enough working with the {CMK} user interface and no longer need the welcome page as your start page, you can change the start page.
At the bottom of the welcome page, you will find the checkbox [.guihint]#Show welcome page on start#.
By deselecting this checkbox, the start page will be changed to the default xref:glossar#dashboard[dashboard].
You can track the corresponding change by selecting [.guihint]#User profile > Edit profile# in the [.guihint]#User# menu of the navigation bar.
The option [.guihint]#Start URL to display in main frame# is now set to [.guihint]#Default start URL#.
By selecting [.guihint]#Welcome page#, you can restore the welcome page as the start page at any time.
