= All files and directories
:title: All files and directories
:description: Reference index of all files and directories mentioned across the {CMK} documentation, organized by location: {CMK} server, monitored Linux hosts, and monitored Windows hosts.

This article is a consolidated reference of all files and directories discussed across the {CMK} documentation.
Paths on the {CMK} server that begin with `~/` are relative to the home directory of the OMD site user (e.g., `/omd/sites/mysite/`).
Paths listed without a `~/` prefix in the xref:ec#[Event Console] and xref:snmp#[SNMP] articles are also relative to the site home directory and are reproduced here exactly as they appear in those articles.


[#cmk-server]
== Paths on the {CMK} server

=== Files

|`~/bin/cmk-piggyback`
|Helper script with analysis tools for piggyback features. Display information about its usage with `cmk-piggyback -h`.
|xref:piggyback#[Piggyback]

|Script command for simplifying issuing of queries and commands to the Unix-Socket in Livestatus.

|`~/etc/password_store.secret`

|`~/tmp/run/mkeventd/events`
|A named pipe where you can write messages directly with `echo` or other commands to pass them to the EC. Make sure that only one application writes to this pipe at any time, otherwise the message texts may get mixed.
|xref:ec#[Event Console]

|`~/tmp/run/mkeventd/eventsocket`
|A Unix socket that does the same job as the pipe, but allows multiple applications to write to it simultaneously.
|xref:ec#[Event Console]

|`~/tmp/run/mkeventd/pid`
|The current process ID of the event daemon while it is running.
|xref:ec#[Event Console]

|`~/tmp/run/mkeventd/status`

|xref:ec#[Event Console]

|xref:availability#[Availability]

|`~/var/check_mk/core/history`
|Current log file for the monitoring history in the CMC.
|xref:availability#[Availability], xref:alert_handlers#[Alert handlers], xref:livestatus#[Livestatus]

|`~/var/check_mk/stored_passwords`
|The password store file containing the obfuscated passwords.
|xref:password_store#[Password store]

|`~/var/check_mk/web/[owner_id]/user_reports.mk`
|These files store the customized and newly created reports of the respective [.guihint]#Owner# of the reports.
|xref:reporting#[Reports]

|`~/var/log/agent-registration.log`
|Log file for the agent registration. You can control the verbosity of the messages in 6 levels via [.guihint]#Setup > General > Global settings > User interface > Logging > Agent registration#.
|xref:hosts_autoregister#[Autoregister hosts]

|`~/var/log/alerts.log`
|Log file with all events relevant to the alert handler (logged by the alert helper).
|xref:alert_handlers#[Alert handlers]

|`~/var/log/cmc.log`
|The CMC's log file, in which the availability cache's statistics, RRD error messages, notification information, and query/command history can be viewed.
|xref:availability#[Availability], xref:graphing#[Graphing], xref:notifications#[Notifications], xref:alert_handlers#[Alert handlers], xref:livestatus#[Livestatus]

|`~/var/log/dcd.log`
|Log file of the Dynamic Configuration Daemon (DCD).
|xref:dcd#[Dynamic host configuration]

|`~/var/log/liveproxyd.log`
|Log file for the Livestatus proxies.
|xref:distributed_monitoring#[Distributed monitoring]

|`~/var/log/liveproxyd.state`
|The current state of the Livestatus proxies in a readable form. This file is updated every 5 seconds.
|xref:distributed_monitoring#[Distributed monitoring]

|`~/var/log/mknotifyd.log`
|Log file for the notification spooler.
|xref:distributed_monitoring#[Distributed monitoring]

|`~/var/log/mknotifyd.state`
|The current state of the notification spooler in a readable form. This file is updated every 20 seconds.
|xref:distributed_monitoring#[Distributed monitoring]

|`~/var/log/mkotifyd.log`
|The notification spooler's log file.
|xref:notifications#[Notifications]

|`~/var/log/mkotifyd.state`
|The current state of the notification spooler. This is primarily relevant for xref:distributed_monitoring#notifications[notifications in distributed environments].
|xref:notifications#[Notifications]

|`~/var/log/nagios.log`
|The Nagios core's log file.
|xref:livestatus#[Livestatus]

|`~/var/log/notify.log`
|The notification module's log file.
|xref:notifications#[Notifications], xref:distributed_monitoring#[Distributed monitoring]

|`~/var/log/rrdcached.log`
|Log file of the RRD cache daemon.
|xref:graphing#[Graphing]

|`~/var/log/web.log`
|Log file of the user interface. Here you will find error messages regarding authentication and LDAP connections.
|xref:wato_user#[User management], xref:ldap#[LDAP integration]

|`~/var/nagios/debug.log`
|The Nagios debug log file. Switch on the debug messages in `~/etc/nagios/nagios.d/logging.cfg` using the `debug_level` variable.
|xref:notifications#[Notifications]

|`~/var/nagios/nagios.log`
|The current log file for Nagios' monitoring history.
|xref:availability#[Availability]

|`~/version`
|Symbolic link to the installation of the {CMK} version used by this site.
|xref:update#[Updates and upgrades]

|`/omd/sites`
|In this directory, for every site there is a home directory containing its configuration files and variable data. This data belongs to the site's user, and can be changed by configuration and operations.
|xref:update#[Updates and upgrades]

|`/omd/versions`
|In this directory a subdirectory exists for every installed {CMK} version. The files belong to `root` and are never changed.
|xref:update#[Updates and upgrades]

|`/usr/bin/omd`
|Management command for {CMK} sites. This is a symbolic link to the default version's `bin` directory.
|xref:update#[Updates and upgrades]

|`etc/check_mk/mkeventd.d/wato/global.mk`
|The global settings of the Event Console in Python syntax.
|xref:ec#[Event Console]

|`etc/check_mk/mkeventd.d/wato/rules.mk`
|All your configured rule packages and rules in Python syntax.
|xref:ec#[Event Console]

|`tmp/run/mkeventd/events`
|A named pipe where you can write messages directly with `echo` or other commands to pass them to the EC.
|xref:ec#[Event Console]

|`tmp/run/mkeventd/eventsocket`
|A Unix socket that does the same job as the pipe, but allows multiple applications to write to it simultaneously.
|xref:ec#[Event Console]

|`tmp/run/mkeventd/pid`
|The current process ID of the event daemon while it is running.
|xref:ec#[Event Console]

|`tmp/run/mkeventd/status`
|A Unix socket that allows querying the current status and sending commands.
|xref:ec#[Event Console]

|`var/log/mkeventd.log`
|Log file of the Event Console.
|xref:ec#[Event Console]

|`var/mkeventd/status`
|Complete current state of the Event Console. This mainly includes all currently open events. In the case of a misconfiguration that leads to a lot of open events, this file can become huge and drastically reduce the performance of the EC.
|xref:ec#[Event Console]

=== Directories

[cols="45,40,~",options="header"]
|===
|Path |Description |Article(s)

|`~/etc/`
|The site's configuration files are stored in this directory.
|xref:omd_basics#[Site administration]

|`~/etc/check_mk/conf.d/`
|The configuration of hosts and rules for {CMK}. Store configuration files generated by CMCDump here.
|xref:distributed_monitoring#[Distributed monitoring]

|`~/etc/check_mk/conf.d/wato/`
|In this directory the folder structure in the Setup below the [.guihint]#Main# folder is mapped by a subdirectory structure. If a folder is created in the GUI, a new directory is also created in the file system.

|Here you will find all plug-ins that are delivered with {CMK}. No distinction is made between plug-ins that are written by monitoring-plugins.org and those that are written specifically for {CMK}.
|xref:active_checks#[Active checks]

|`~/local/bin/`
|The repository for own programs and scripts that should be in a search path, and which can be directly executed without specifying the path. If a program is in `~/bin/` as well as in `~/local/bin/`, the latter has priority.
|xref:datasource_programs#[Datasource programs], xref:devel_special_agents#[Special agent development]

|`~/local/lib/check_mk/base/cee/plugins/bakery/`
|Directory for the Bakery plug-in.

|xref:mkp_viewables#[MKP extensions]

|`~/local/lib/check_mk/gui/plugins/views/`
|Depository for views cloned as extensions.
|xref:mkp_viewables#[MKP extensions]

|`~/var/check_mk/autochecks/`
|For services found by the service discovery. These are always stored locally on the remote site.
|xref:distributed_monitoring#[Distributed monitoring]
